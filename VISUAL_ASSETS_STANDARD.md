# Золотой Стандарт Wild Rose: Визуальные Ассеты (ГОСТ)

Этот документ описывает правила именования и организации фотографий для сайта **Wild Rose Jewel**. Соблюдение этих правил гарантирует стабильную работу сайта и легкость в управлении контентом.

---

## 1. Структура папок (Иерархия)

Все фотографии товаров должны находиться строго в этой структуре:

`assets/products/[сезон]/[ID]_[название-коллекции]/[категория]/`

- **[сезон]**: `spring`, `summer`, `autumn`, `winter`, `symbol` (символы/база).
- **[ID]**: 4-значный номер коллекции (например, `0021`).
- **[название-коллекции]**: Латиница, через дефис (например, `pearl-sea`).
- **[категория]**: `necklaces`, `rings`, `earrings`, `bracelets`, `sets`.

**Пример пути:**
`assets/products/summer/0021_pearl-sea/necklaces/`

### 🔹 1.1. Допустимые категории (Только в мн. числе):
- `necklaces` — (колье, ожерелья, цепочки)
- `rings` — (кольца)
- `earrings` — (серьги, каффы)
- `bracelets` — (браслеты)
- `sets` — (наборы, сеты)

---

## 2. Именование файлов (Формула)

Имя файла наследуется от ID изделия. Это позволяет мгновенно находить нужные фото.

**Формула:** `[ID-колекции]-[номер-товара]_[тип].[расширение]`

### 🔹 Важно! Смысл номера товара:
- Одна коллекция (папка) может содержать много разных товаров.
- Каждый **отдельный** товар в каталоге должен иметь **свой уникальный номер** после дефиса.
- Мы используем **жесткую привязку** индекса к типу изделия для идеального порядка:
    1. `-1` — **Necklaces** (Колье, цепочки)
    2. `-2` — **Bracelets** (Браслеты)
    3. `-3` — **Earrings** (Серьги)
    4. `-4` — **Rings** (Кольца)
    5. `-5` — **Sets** (Наборы, сеты)

### 🔹 2.1. Реальный пример (Коллекция 0001_bablgam):
Если в вашей папке коллекции лежат разные типы изделий, подписывайте их так:
- **Колье** (0001-1) → лежит в `necklaces/` → файлы называются `0001-1_main.jpg`
- **Браслет** (0001-2) → лежит в `bracelets/` → файлы называются `0001-2_main.jpg`
- **Серьги** (0001-3) → лежат в `earrings/` → файлы называются `0001-3_main.jpg`
- **Кольцо** (0001-4) → лежит в `rings/` → файлы называются `0001-4_main.jpg`
- **Сет** (0001-5) → лежит в `sets/` → файлы называются `0001-5_main.jpg`

---

### 🔹 Компоненты имени:
1. **[ID-колекции]-[номер-товара]**: Это уникальный ID изделия (например, `0021-1`).
2. **[тип]**:
    - `main` — Основное фото (на белом/чистом фоне). Используется в сетке каталога.
    - `look` — Имиджевое фото (на модели). Используется для функции «Образ».
    - `det-NN` — Детали/Макро для галереи «Подробности» (например, `det-01`, `det-02`).
    - `side` — Вид сбоку или с другого ракурса.
    - `set` — Фото всего набора (если применимо).

### ✅ Примеры правильных имен для одного товара (Галерея):
- `0021-1_main.jpg` (Главное фото)
- `0021-1_look.jpg` (На модели)
- `0021-1_det-01.jpg` (Первое фото в галерее подробностей)
- `0021-1_det-02.jpg` (Второе фото в галерее подробностей)
- `0021-1_side.jpg` (Вид сбоку)

## 3. Критические правила (Важно!)

> [!CAUTION]
> **Никакой кириллицы в именах файлов и папок!**
> Кириллица в ссылках может «ломаться» в разных браузерах и мессенджерах. Используйте только английский язык и дефисы вместо пробелов.

> [!IMPORTANT]
> **Золотое сечение оптимизации (Wild Rose Quality)**
> Для сохранения премиальной детализации при быстрой загрузке используйте:
> - **Формат:** WebP (рекомендуется) или JPEG.
> - **Качество (Quality):** 80-85%.
> - **Размер (Resize):** 1600px по длинной стороне (этого достаточно для 4K и Retina-дисплеев).
> - **Итоговый вес:** 400-700 Кб на один файл.

## 4. Как внедрять в код (`products-data.js`)

При добавлении нового товара просто копируйте его ID в пути:

```javascript
{
    id: "0021-1",
    collection: "Жемчужина моря",
    category: "necklaces",
    mainImage: "assets/products/summer/0021_pearl-sea/necklaces/0021-1_main.jpg",
    lookModel: "assets/products/summer/0021_pearl-sea/necklaces/0021-1_look.jpg"
}
```

*Соблюдение этого стандарта — залог профессионального и долговечного сайта.* 🌹
